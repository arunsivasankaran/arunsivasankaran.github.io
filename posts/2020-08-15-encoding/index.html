(WIP) Character encodings

2020-08-15
binary ascii , unicode , utf-8

ASCII

I started attending an online class called the Bradfield Computer Science Intensive. I can tell you that I am learning a lot of new and exciting stuff. Stuff I should have learned in college, but didn’t. I might write a post about that someday. But for now Ill try and get straight to the point.

Computers only speak one language. Its binary. The only 2 alphabets are [0, 1]. You can also think of them as [On, Off], [Up, Down]..[Good, Evil]. I wanted to write a short funny blurb about how binary was invented by a soldier who lost all but 2 of his fingers fighting for king arthur. But I realized I am a horrible story teller. So…

Anyway. People, unlike computers, normally use languages with many alphabets. Like english with 26, or Tamil with 247 or Klingon - also with 26.

So, we needed computers to be able to represent these alphabets - otherwise whats the point of computers? So people came up with a variety of techniques to store and represent english alphabets in binary. So, we all know we can use binary to represent numbers. Its pretty straight forward. So all we have to do is assign a number to each of the alphabets, and then voila. If we decide a is 0, b is 1 and c is 2, then abc is just (1, 2, 3) in binary - which translates to [00000000, 00000001, 00000010]. This way, we can encode all english characters, numbers and pretty much anything else, made up or real as numbers and so, as binary.

ASCII or “American Standard Code for Information Interchange” is a such a character encodings map.

With out a standard, like ASCII, people can make up their own character encodings. And these encodings wont be compatible. So we would lose interoperability. ASCII solves that problem. As long as one encodes text in ASCII, we can be sure that the text will be interpreted properly by anyone else using the ASCII standard. There - thats one problems solved.

The ASCII standard, encodes a character as a 7 bit integer. With 7 bits, we can represent 128 characters. No more. Now that is more than enough for to represent all of english, lower case and upper case, all numbers 0 - 7 and a still have some space left over. But not a lot.

Even though ASCII characters use only 7 bits, we use all 8 bits in a byte to represent them. This means the 8th bit is wasted. Normally its just zeroed out, but sometimes, the 8th bit is used as a parity bit used for error detection. We will talk about this in a different post.

The ASCII standard works well for Americans. It might also work pretty well for most latin derived languages. But what about tamil? If you recall, there are 247 alphabets in the Tamil language. 7 bits is NOT enough. And what about other languages?
